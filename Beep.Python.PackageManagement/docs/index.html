Beep.Python.PackageManagement

Enterprise Python package management system for .NET applications with dependency resolution, virtual environment integration, and comprehensive package lifecycle management

Installation Quick Start API Reference

?? Overview

Beep.Python.PackageManagement provides a comprehensive solution for managing Python packages in .NET environments. It offers enterprise-grade package management with dependency resolution, version control, and integration with virtual environments and CI/CD pipelines.

Key Features
  • Advanced Package Management: Install, update, uninstall with dependency resolution
  • Requirements Management: Parse and manage requirements.txt files
  • Package Categories: Organize packages by categories and project types
  • Package Sets: Define and manage collections of related packages
  • Dependency Analysis: Analyze package dependencies and conflicts
  • Virtual Environment Integration: Seamless integration with Python virtual environments
  • Enterprise Features: Proxy support, private repositories, security scanning

??? Architecture Components

PythonPackageManager

Core package management orchestrator handling installation, updates, and environment coordination

PackageCategoryManager

Organizes packages into logical categories like ML, Data Science, Web Development, etc.

PackageSetManager

Manages predefined package collections for specific use cases and project templates

RequirementsFileManager

Handles requirements.txt parsing, generation, and dependency file management

PackageOperationManager

Executes package operations with progress tracking, error handling, and rollback capabilities

?? Quick Start

Basic Package Management

using Beep.Python.PackageManagement;
using TheTechIdea.Beep.Container.Services;

// Initialize the package manager
var beepService = new BeepService();
var packageManager = new PythonPackageManager(beepService);

// Configure for a specific virtual environment
packageManager.SetVirtualEnvironment("my-project-env");

// Install packages
await packageManager.InstallPackageAsync("pandas", "2.1.0");
await packageManager.InstallPackageAsync("numpy");
await packageManager.InstallPackageAsync("scikit-learn", ">=1.3.0");

// Install from requirements file
await packageManager.InstallFromRequirementsAsync("requirements.txt");

// Check installed packages
var installedPackages = await packageManager.GetInstalledPackagesAsync();
foreach (var package in installedPackages)
{
    Console.WriteLine($"{package.Name} {package.Version}");
}

// Update packages
await packageManager.UpdatePackageAsync("pandas");
await packageManager.UpdateAllPackagesAsync();

// Uninstall packages
await packageManager.UninstallPackageAsync("old-package");

    // Package Removal

    public async Task<bool> UninstallPackagesAsync(IEnumerable<string> packages);
    
    // Package Information
    public async Task<List<PackageInfo>> GetInstalledPackagesAsync();
    public async Task<PackageInfo> GetPackageInfoAsync(string packageName);

RequirementsFileManager

Handles parsing, generation, and management of requirements.txt and other dependency files.

Requirements File Operations

var requirementsManager = new RequirementsFileManager(beepService);

// Parse requirements file
var requirements = await requirementsManager.ParseRequirementsFileAsync("requirements.txt");
foreach (var req in requirements)
{
    Console.WriteLine($"{req.Name} {req.VersionSpecifier} ({req.Extras})");
}

// Generate requirements from current environment
await requirementsManager.GenerateRequirementsFileAsync(
    "generated-requirements.txt", 
    includeVersions: true,
    includeDependencies: false);

// Create development requirements
var devRequirements = new[]
{
    "pytest>=7.0.0",
    "black>=22.0.0",
    "flake8>=5.0.0",
    "mypy>=0.991",
    "pre-commit>=2.20.0"
};

await requirementsManager.CreateRequirementsFileAsync("dev-requirements.txt", devRequirements);

// Merge multiple requirements files
await requirementsManager.MergeRequirementsFilesAsync(
    new[] { "requirements.txt", "dev-requirements.txt" },
    "all-requirements.txt");

// Validate requirements file
var validation = await requirementsManager.ValidateRequirementsFileAsync("requirements.txt");
if (!validation.IsValid)
{
    foreach (var error in validation.Errors)
    {
        Console.WriteLine($"Error: {error.Message} at line {error.LineNumber}");
    }
}

// Pin package versions
await requirementsManager.PinVersionsInRequirementsFileAsync(
    "requirements.txt", 
    "requirements-pinned.txt");

// Update requirements file
await requirementsManager.UpdateRequirementsFileAsync("requirements.txt", new[]
{
    new PackageRequirement { Name = "pandas", VersionSpecifier = ">=2.0.0" },
    new PackageRequirement { Name = "numpy", VersionSpecifier = ">=1.24.0" }
});

var securityConfig = new PackageSecurityConfig

    EnableVulnerabilityScanning = true,

}

Dependency Analysis

Advanced dependency resolution and conflict detection capabilities.

Dependency Management

// Analyze dependency tree
var dependencyTree = await packageManager.GetDependencyTreeAsync("tensorflow");
PrintDependencyTree(dependencyTree);

// Check for conflicts
var conflicts = await packageManager.CheckDependencyConflictsAsync();
foreach (var conflict in conflicts)
{
    Console.WriteLine($"Conflict: {conflict.Package1} requires {conflict.Dependency} {conflict.Version1}");
    Console.WriteLine($"         {conflict.Package2} requires {conflict.Dependency} {conflict.Version2}");
}

// Resolve conflicts automatically
var resolution = await packageManager.ResolveDependencyConflictsAsync();
if (resolution.Success)
{
    Console.WriteLine("Conflicts resolved successfully:");
    foreach (var change in resolution.Changes)
    {
        Console.WriteLine($"  {change.Action}: {change.Package} {change.Version}");
    }
}

// Generate dependency graph
await packageManager.GenerateDependencyGraphAsync("dependency-graph.dot", GraphFormat.Graphviz);

?? Enterprise Integration

CI/CD Pipeline Integration

// Pipeline package management
public class PipelinePackageManager
{
    private readonly PythonPackageManager _packageManager;
    
    public async Task SetupPipelineEnvironmentAsync(string pipelineConfig)
    {
        // Load pipeline configuration
        var config = JsonSerializer.Deserialize<PipelineConfig>(pipelineConfig);
        
        // Create isolated environment
        await _packageManager.CreateVirtualEnvironmentAsync(config.EnvironmentName);
        _packageManager.SetVirtualEnvironment(config.EnvironmentName);
        
        // Install base packages
        await _packageManager.InstallFromRequirementsAsync(config.RequirementsFile);
        
        // Install test dependencies
        if (config.IncludeTestDependencies)
        {
            await _packageManager.InstallFromRequirementsAsync("test-requirements.txt");
        }
        
        // Validate environment
        var validation = await ValidateEnvironmentAsync(config.ValidationRules);
        if (!validation.IsValid)
        {
            throw new InvalidOperationException($"Environment validation failed: {validation.ErrorMessage}");
        }
    }
    
    public async Task<EnvironmentSnapshot> CreateEnvironmentSnapshotAsync()
    {
        return new EnvironmentSnapshot
        {
            Timestamp = DateTime.UtcNow,
            Packages = await _packageManager.GetInstalledPackagesAsync(),
            PythonVersion = await _packageManager.GetPythonVersionAsync(),
            EnvironmentPath = _packageManager.CurrentEnvironmentPath,
            RequirementsHash = await ComputeRequirementsHashAsync()
        };
    }
}

? Best Practices

Package Management Best Practices
  • Version Pinning: Always pin package versions in production requirements files
  • Virtual Environments: Use isolated environments for different projects
  • Security Scanning: Regularly scan packages for known vulnerabilities
  • Dependency Management: Keep dependencies minimal and well-documented
  • Testing: Test package installations in clean environments
Common Pitfalls
  • Unpinned Dependencies: Avoid using loose version constraints in production
  • System-wide Installation: Never install packages globally in production systems
  • Ignored Conflicts: Always resolve dependency conflicts before deployment
  • Missing Requirements: Ensure all dependencies are captured in requirements files
