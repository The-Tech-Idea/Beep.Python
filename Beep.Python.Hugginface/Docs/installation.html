??? System Requirements

.NET Framework

Required: .NET 6.0 or higher

  • .NET 6.0 (LTS)
  • .NET 7.0
  • .NET 8.0 (LTS)
  • .NET 9.0 (Latest)

Operating Systems

Supported Platforms:

  • Windows 10/11 (x64, ARM64)
  • macOS 10.15+ (x64, ARM64)
  • Linux (Ubuntu 18.04+, RHEL 8+)
  • Docker containers

Python Environment

Required: Python 3.8 or higher

  • Python 3.8+ installed
  • pip package manager
  • Virtual environment support
  • Administrative privileges (for package installation)

Hardware Requirements

Minimum specifications:

  • 4 GB RAM (8 GB recommended)
  • 2 GB free disk space
  • Internet connection (for AI API calls)
  • GPU optional (for local models)

?? NuGet Installation

Method 1: Package Manager UI

Visual Studio Package Manager

  1. Right-click on your project in Visual Studio
  2. Select "Manage NuGet Packages..."
  3. Click on the "Browse" tab
  4. Search for "Beep.Python.AI.Transformers"
  5. Click "Install" on the package

Method 2: Package Manager Console

PowerShell Command

Install-Package Beep.Python.AI.Transformers

Method 3: .NET CLI

Command Line Installation

# Add package to your project
dotnet add package Beep.Python.AI.Transformers

# Restore packages
dotnet restore

Method 4: PackageReference

Add to your .csproj file

<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Beep.Python.AI.Transformers" Version="1.0.0" />
  </ItemGroup>
</Project>

?? Python Environment Setup

?? Tip: The library automatically detects and configures Python environments, but manual setup provides better control for production deployments.

1. Python Installation Verification

Check Python Installation

# Check Python version
python --version
# or
python3 --version

# Check pip version
pip --version

# Verify Python.NET compatibility
python -c "import sys; print(f'Python {sys.version}')"

2. Virtual Environment Setup (Recommended)

Create Virtual Environment

# Create virtual environment
python -m venv transformers-env

# Activate environment (Windows)
transformers-env\Scripts\activate

# Activate environment (macOS/Linux)
source transformers-env/bin/activate

# Verify activation
which python
pip list

Manual Python Environment Configuration

using Beep.Python.Model;

Basic Verification Test

Simple Test Application

using Beep.Python.AI.Transformers;
using Beep.Python.Model;
using Microsoft.Extensions.DependencyInjection;

public class Program
{
    public static async Task Main(string[] args)
    {
        // Setup dependency injection
        var services = new ServiceCollection();
        services.AddSingleton<IPythonRunTimeManager, PythonNetRunTimeManager>();
        services.AddSingleton<IPythonCodeExecuteManager, PythonCodeExecuteManager>();
        
        var serviceProvider = services.BuildServiceProvider();
        
        // Test basic functionality
        var runtimeManager = serviceProvider.GetRequiredService<IPythonRunTimeManager>();
        var executeManager = serviceProvider.GetRequiredService<IPythonCodeExecuteManager>();
        
        // Verify Python runtime
        var isInitialized = await runtimeManager.InitializeAsync();
        Console.WriteLine($"Python Runtime Initialized: {isInitialized}");
        
        // Test transformer pipeline creation
        try
        {
            var pipeline = new HuggingFaceTransformerPipeline(runtimeManager, executeManager);
            Console.WriteLine("? Transformer pipeline created successfully!");
            
            var config = new TransformerPipelineConfig
            {
                TaskType = TransformerTask.TextGeneration
            };
            
            var initialized = await pipeline.InitializeAsync(config);
            Console.WriteLine($"? Pipeline initialized: {initialized}");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"? Error: {ex.Message}");
        }
    }
}

Environment Validation

Comprehensive Environment Check

public static async Task ValidateEnvironmentAsync()
{
    Console.WriteLine("?? Validating Beep.Python.AI.Transformers Environment...\n");
    
    // 1. Check .NET version
    var dotnetVersion = Environment.Version;
    Console.WriteLine($"? .NET Runtime: {dotnetVersion}");
    
    // 2. Check Python availability
    var runtimeManager = new PythonNetRunTimeManager(null);
    var pythonInstallations = runtimeManager.PythonInstallations;
    
    if (pythonInstallations.Any())
    {
        Console.WriteLine($"? Python Installations Found: {pythonInstallations.Count}");
        foreach (var installation in pythonInstallations)
        {
            Console.WriteLine($"   - {installation.Name}: {installation.RuntimePath}");
        }
    }
    else
    {
        Console.WriteLine("? No Python installations found");
        return;
    }
    
    // 3. Test package imports
    var testPackages = new[] { "numpy", "requests", "json" };
    foreach (var package in testPackages)
    {
        try
        {
            var result = await executeManager.ExecuteCodeAsync($"import {package}");
            Console.WriteLine($"? Package '{package}': Available");
        }
        catch
        {
            Console.WriteLine($"? Package '{package}': Missing");
        }
    }
    
    Console.WriteLine("\n?? Environment validation complete!");
}

?? Troubleshooting

Common Issues and Solutions

?? Python Not Found

If you get "Python not found" errors:

  • Ensure Python is in your system PATH
  • Restart your IDE/command prompt after Python installation
  • Specify the exact Python path in configuration
  • Check that Python version is 3.8 or higher
?? Package Import Errors

If Python packages fail to import:

  • Activate the correct virtual environment
  • Install packages with pip install [package]
  • Check for conflicting Python installations
  • Verify package compatibility with your Python version
?? Performance Issues

For optimal performance:

  • Use virtual environments for isolation
  • Install only required provider packages
  • Configure appropriate timeouts
  • Monitor memory usage with large models

Docker Installation

Dockerfile for Transformers Applications

# Use official .NET runtime image
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
EXPOSE 80

# Install Python and dependencies
RUN apt-get update && apt-get install -y \
    python3 \
    python3-pip \
    python3-venv \
    && rm -rf /var/lib/apt/lists/*

# Create Python virtual environment
RUN python3 -m venv /opt/transformers-env
ENV PATH="/opt/transformers-env/bin:$PATH"

# Install Python packages
RUN pip install --upgrade pip && \
    pip install pythonnet>=3.0.3 && \
    pip install numpy requests

# Copy and build .NET application
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ["YourApp.csproj", "."]
RUN dotnet restore "YourApp.csproj"
COPY . .
RUN dotnet build "YourApp.csproj" -c Release -o /app/build

FROM build AS publish
RUN dotnet publish "YourApp.csproj" -c Release -o /app/publish

FROM base AS final
WORKDIR /app
COPY --from=publish /app/publish .
ENTRYPOINT ["dotnet", "YourApp.dll"]
