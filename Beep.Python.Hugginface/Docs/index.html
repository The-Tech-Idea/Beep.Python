Beep.Python.AI.Transformers

Enterprise-grade multi-provider transformer pipeline library for .NET 6, 7, 8, and 9

Overview

The Beep.Python.AI.Transformers library provides a unified, enterprise-grade abstraction for working with transformer models from multiple AI service providers. It offers seamless integration with HuggingFace, OpenAI, Azure OpenAI, Google AI, Anthropic, and many other providers through a consistent, type-safe API.

Key Benefits
  • Provider Agnostic - Switch between AI providers with minimal code changes
  • Enterprise Security - Built-in authentication, rate limiting, and compliance
  • Session Management - Multi-user isolation with virtual environments
  • Multimodal Support - Text, image, audio, and video processing
  • Production Ready - Monitoring, analytics, and error handling

?? Code Examples

Basic Usage - OpenAI Integration

using Beep.Python.AI.Transformers;

// Configure OpenAI connection
var openAIConfig = new OpenAIConnectionConfig
{
    ApiKey = "sk-your-api-key",
    TimeoutSeconds = 60
};

// Create pipeline with connection
var pipeline = TransformerPipelineFactory.CreateOpenAIPipeline(
    pythonRunTimeManager,
    executeManager,
    openAIConfig
);

// Initialize and load model
await pipeline.InitializeAsync(new TransformerPipelineConfig
{
    TaskType = TransformerTask.TextGeneration
});

await pipeline.LoadModelAsync(new TransformerModelInfo
{
    Name = "gpt-4",
    Source = TransformerModelSource.OpenAI
}, TransformerTask.TextGeneration);

// Generate text
var result = await pipeline.GenerateTextAsync("Write a short story about AI");
Console.WriteLine(result.Data); // AI-generated response

Multi-Provider Enterprise Setup

// Configure multiple providers
var providers = new Dictionary<TransformerModelSource, TransformerConnectionConfig>
{
    [TransformerModelSource.OpenAI] = new OpenAIConnectionConfig { ApiKey = "..." },
    [TransformerModelSource.Azure] = new AzureOpenAIConnectionConfig { Endpoint = "..." },
    [TransformerModelSource.Anthropic] = new AnthropicConnectionConfig { ApiKey = "..." }
}

// Create enterprise pipeline with session management
var enterprisePipeline = TransformerPipelineFactory.CreateSessionAwarePipelineWithConnection(
    TransformerModelSource.OpenAI,
    pythonRunTimeManager,
    executeManager,
    username: "alice",
    connectionConfig: providers[TransformerModelSource.OpenAI]
);

// Use the same API regardless of provider
var result = await enterprisePipeline.GenerateTextAsync("Explain quantum computing");

Multimodal AI - Text to Image

// Create multimodal pipeline
var multimodalPipeline = new MultimodalTransformerPipeline(
    pythonRunTimeManager, 
    executeManager
);

// Initialize with configuration
await multimodalPipeline.InitializeAsync(new MultimodalPipelineConfig
{
    PreloadPipelines = new List<MultimodalTask> 
    { 
        MultimodalTask.TextToImage,
        MultimodalTask.ImageToText
    }
});

// Generate image from text
var imageResult = await multimodalPipeline.GenerateImageFromTextAsync(
    "A futuristic cityscape at sunset",
    new TextToImageParameters 
    { 
        Width = 1024, 
        Height = 768,
        Style = "photorealistic"
    }
);

if (imageResult.Success)
{
    Console.WriteLine($"Image generated: {imageResult.Data.ImageData}");
}

??? Architecture

The library follows a layered architecture with clear separation of concerns:

Application Layer

Your application code using the transformer pipelines with type-safe APIs

Pipeline Layer

ITransformerPipeLine interface and provider-specific implementations

Security Layer

Connection management, authentication, session handling, and compliance

Runtime Layer

Python.NET integration with virtual environment and session management

Provider Layer

AI service providers (OpenAI, Azure, HuggingFace, Anthropic, etc.)

Data Layer

Model information, configurations, and result data structures
