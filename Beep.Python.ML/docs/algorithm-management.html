?? Classification Algorithms

RandomForestClassifier

Ensemble method combining multiple decision trees for robust classification with feature importance ranking.

Optimal Parameters

var parameters = new Dictionary<string, object>
{
    ["n_estimators"] = 100,        // Number of trees (50-200)
    ["max_depth"] = 10,            // Maximum tree depth (5-20)
    ["min_samples_split"] = 2,     // Min samples to split (2-10)
    ["min_samples_leaf"] = 1,      // Min samples per leaf (1-5)
    ["max_features"] = "sqrt",     // Features per split ("sqrt", "log2", "auto")
    ["bootstrap"] = true,          // Bootstrap sampling
    ["random_state"] = 42,         // Reproducibility
    ["class_weight"] = "balanced", // Handle imbalanced classes
    ["n_jobs"] = -1               // Use all CPU cores
};

mlManager.TrainModel("rf_model", MachineLearningAlgorithm.RandomForestClassifier, 
                    parameters, features, "target");
Use Cases
  • Feature Importance: Excellent for understanding feature contributions
  • Robust to Outliers: Handles noisy data well
  • No Scaling Required: Works with raw numerical features
  • Mixed Data Types: Handles numerical and categorical features

GradientBoostingClassifier

Sequential ensemble method that builds models to correct errors of previous models.

Optimal Parameters

var parameters = new Dictionary<string, object>
{
    ["n_estimators"] = 150,        // Number of boosting stages (100-300)
    ["learning_rate"] = 0.1,       // Step size shrinkage (0.01-0.3)
    ["max_depth"] = 6,             // Individual tree depth (3-10)
    ["min_samples_split"] = 2,     // Min samples to split (2-10)
    ["min_samples_leaf"] = 1,      // Min samples per leaf (1-5)
    ["subsample"] = 0.8,           // Fraction of samples per tree (0.5-1.0)
    ["max_features"] = "sqrt",     // Features per split
    ["random_state"] = 42,         // Reproducibility
    ["validation_fraction"] = 0.1  // Early stopping validation
};

mlManager.TrainModel("gb_model", MachineLearningAlgorithm.GradientBoostingClassifier, 
                    parameters, features, "target");
Performance Considerations
  • Training Time: Slower than Random Forest due to sequential nature
  • Overfitting Risk: Monitor validation performance carefully
  • Hyperparameter Sensitive: Requires careful tuning
  • Memory Usage: Can be memory intensive with large datasets

LogisticRegression

Linear model for binary and multiclass classification with interpretable coefficients.

Optimal Parameters

var parameters = new Dictionary<string, object>
{
    ["C"] = 1.0,                   // Regularization strength (0.001-100)
    ["penalty"] = "l2",            // Regularization type ("l1", "l2", "elasticnet")
    ["solver"] = "liblinear",      // Optimization algorithm
    ["max_iter"] = 1000,           // Maximum iterations (100-5000)
    ["class_weight"] = "balanced", // Handle imbalanced classes
    ["random_state"] = 42,         // Reproducibility
    ["multi_class"] = "ovr"        // Multi-class strategy ("ovr", "multinomial")
};

mlManager.TrainModel("lr_model", MachineLearningAlgorithm.LogisticRegression, 
                    parameters, features, "target");
Best Practices
  • Feature Scaling: Always scale features for optimal performance
  • Regularization: Use L1 for feature selection, L2 for stability
  • Interpretability: Coefficients provide direct feature importance
  • Fast Training: Excellent for large datasets with quick iterations

?? Algorithm Selection Guide

Classification Problems

  • Binary Classification: LogisticRegression, SVC, RandomForestClassifier
  • Multi-class: RandomForestClassifier, GradientBoostingClassifier
  • Text Classification: MultinomialNB, SVC with linear kernel
  • Imbalanced Data: RandomForestClassifier with class weights
  • High Dimensions: SVC, LogisticRegression with L1

Regression Problems

  • Linear Relationships: LinearRegression, RidgeRegression
  • Non-linear: RandomForestRegressor, GradientBoostingRegressor
  • Feature Selection: LassoRegression, ElasticNet
  • Large Datasets: HistGradientBoostingRegressor
  • Robust to Outliers: RandomForestRegressor

Grid Search for Parameter Tuning

public Dictionary<string, object> OptimizeRandomForestParameters()
{
    var parameterGrids = new[]
    {
        new Dictionary<string, object>
        {
            ["n_estimators"] = 50,
            ["max_depth"] = 5,
            ["min_samples_split"] = 2
        },
        new Dictionary<string, object>
        {
            ["n_estimators"] = 100,
            ["max_depth"] = 10,
            ["min_samples_split"] = 5
        },
        new Dictionary<string, object>
        {
            ["n_estimators"] = 200,
            ["max_depth"] = 15,
            ["min_samples_split"] = 10
        }
    };

    var bestScore = 0.0;
    var bestParameters = parameterGrids[0];

    foreach (var parameters in parameterGrids)
    {
        parameters["random_state"] = 42;
        
        mlManager.TrainModel("temp_model", MachineLearningAlgorithm.RandomForestClassifier, 
                           parameters, features, "target");
        
        var scores = mlManager.GetModelClassificationScore("temp_model");
        
        if (scores.Item2 > bestScore) // F1-score
        {
            bestScore = scores.Item2;
            bestParameters = new Dictionary<string, object>(parameters);
        }
    }

    return bestParameters;
}
Optimization Best Practices
  • Start Simple: Begin with default parameters and gradually tune
  • Cross-Validation: Use cross-validation to validate parameter choices
  • Feature Scaling: Scale features before tuning linear models
  • Early Stopping: Use validation sets to prevent overfitting
  • Computational Budget: Balance accuracy with training time
