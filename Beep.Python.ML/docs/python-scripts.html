?? Template Scripts Overview

Beep.Python.ML uses a revolutionary template-based architecture with over 101 Python scripts that eliminate inline code mixing. All Python operations are executed through parameterized templates managed by the PythonScriptTemplateManager.

?? PythonScriptTemplateManager

The core template management system that loads, caches, and executes Python scripts with parameter substitution.

Template Manager Usage

// All assistant classes use the template manager
public class PythonDataPreprocessingAssistant : BasePythonAssistant
{
    public void StandardizeData(string[] selectedFeatures = null)
    {
        // Get parameterized script from template manager
        var script = PythonScriptTemplateManager.GetScript(PythonScriptType.StandardizeData);
        
        // Substitute parameters
        var parameters = new Dictionary<string, object>
        {
            ["selected_features"] = selectedFeatures ?? new string[0],
            ["feature_list"] = string.Join(",", selectedFeatures ?? new string[0])
        };
        
        var parameterizedScript = SubstituteParameters(script, parameters);
        
        // Execute using session manager
        ExecuteScript(parameterizedScript);
    }
}

Script Template Structure

# Template: standardize_data.py
# Parameters: {selected_features}, {feature_list}

import pandas as pd
from sklearn.preprocessing import StandardScaler

# Load data from session
df = globals()['df']

# Feature selection
selected_features = {selected_features}
if len(selected_features) > 0:
    features_to_scale = selected_features
else:
    features_to_scale = df.select_dtypes(include=['float64', 'int64']).columns.tolist()

# Standardization
scaler = StandardScaler()
df[features_to_scale] = scaler.fit_transform(df[features_to_scale])

# Store results back to session
globals()['df'] = df
globals()['scaler'] = scaler

print(f"? Standardized {len(features_to_scale)} features: {feature_list}")

Best Practices

Script Development Best Practices
  • Error Handling: Always wrap main logic in try-catch blocks
  • Parameter Validation: Validate all parameters before use
  • Session State: Use globals() for session data persistence
  • Progress Reporting: Provide clear success/failure messages
  • Documentation: Include header comments with purpose and parameters
  • Imports: Import only required libraries for performance
  • Data Validation: Always check data exists before processing
Common Pitfalls
  • Missing Parameters: Always provide default values for optional parameters
  • Session Conflicts: Use unique variable names in session globals
  • Memory Management: Clean up large intermediate variables
  • Type Safety: Validate parameter types before use
  • Path Handling: Use absolute paths for file operations

Caching and Optimization

Script Caching Mechanism

public class PythonScriptTemplateManager
{
    private static readonly ConcurrentDictionary<PythonScriptType, string> _scriptCache 
        = new ConcurrentDictionary<PythonScriptType, string>();
    
    public static string GetScript(PythonScriptType scriptType)
    {
        // Check cache first
        if (_scriptCache.TryGetValue(scriptType, out string cachedScript))
        {
            return cachedScript;
        }
        
        // Load script from file
        var scriptPath = GetScriptPath(scriptType);
        var script = File.ReadAllText(scriptPath);
        
        // Cache for future use
        _scriptCache.TryAdd(scriptType, script);
        
        return script;
    }
}
Performance Benefits
  • Script Caching: Scripts loaded once and cached in memory
  • Lazy Loading: Scripts loaded only when needed
  • Parameter Optimization: Efficient parameter substitution
  • Session Reuse: Python session state maintained across calls
  • Minimal Overhead: Template processing optimized for speed
