?? Session Configuration Issues

The most common category of issues relates to session configuration. Here are the symptoms and solutions.

Common Session Errors

Error: "Python environment is not initialized"

InvalidOperationException: Python environment is not initialized
   at Beep.Python.ML.PythonMLManager.TrainModel(...)
?? Solution:
// Always configure session before any ML operations
var mlManager = new PythonMLManager(beepService, pythonRuntime, sessionInfo);

// REQUIRED: Configure session first
bool sessionConfigured = mlManager.ConfigureMLSessionForUser("username");
if (!sessionConfigured)
{
    throw new InvalidOperationException("Failed to configure ML session");
}

// Now you can perform ML operations
var features = mlManager.LoadData("data.csv");

Error: "Session configuration failed"

Failed to configure ML session for user: username
?? Solution:
// Check Python runtime initialization
var pythonRuntime = new PythonNetRunTimeManager(beepService);

// Verify Python installation
if (!pythonRuntime.IsPythonInstalled())
{
    throw new InvalidOperationException("Python is not installed or not accessible");
}

// Configure with proper session info
var sessionInfo = new PythonSessionInfo 
{ 
    Username = "username",
    SessionId = Guid.NewGuid().ToString(),
    CreateTime = DateTime.UtcNow
};

var mlManager = new PythonMLManager(beepService, pythonRuntime, sessionInfo);

// Configure with error checking
try
{
    bool configured = mlManager.ConfigureMLSessionForUser("username");
    if (!configured)
    {
        // Check session status
        var configuredSession = mlManager.GetConfiguredSession();
        var configuredVirtualEnv = mlManager.GetConfiguredVirtualEnvironment();
        
        Console.WriteLine($"Session: {configuredSession?.Username}");
        Console.WriteLine($"Virtual Env: {configuredVirtualEnv?.Name}");
    }
}
catch (Exception ex)
{
    Console.WriteLine($"Session configuration error: {ex.Message}");
}

Error: "Assistant class access failed"

NullReferenceException: Object reference not set to an instance of an object
   at accessing assistant classes
?? Solution:
// CORRECT: Cast to implementation to access assistants
if (mlManager is PythonMLManager manager)
{
    // Configure session first
    if (!manager.IsSessionConfigured())
    {
        manager.ConfigureMLSessionForUser("username");
    }
    
    // Now access assistant classes
    manager.DataPreprocessing.StandardizeData();
    manager.FeatureEngineering.GeneratePolynomialFeatures();
}
else
{
    throw new InvalidOperationException("Failed to cast to PythonMLManager implementation");
}

// INCORRECT: Don't try to access assistants through interface
// mlManager.DataPreprocessing.StandardizeData(); // This will fail

?? File and Data Issues

Common problems with file paths, data loading, and data format issues.

File Path Problems

Error: "File not found"

FileNotFoundException: Data file not found: data.csv
?? Solution:
// Always check file existence before loading
string dataFilePath = "data/customer_data.csv";

if (!File.Exists(dataFilePath))
{
    // Try absolute path
    dataFilePath = Path.GetFullPath(dataFilePath);
    
    if (!File.Exists(dataFilePath))
    {
        // Try alternative locations
        var alternativePaths = new[]
        {
            Path.Combine(Directory.GetCurrentDirectory(), "data", "customer_data.csv"),
            Path.Combine(AppDomain.CurrentDomain.BaseDirectory, "data", "customer_data.csv"),
            @"C:\Data\customer_data.csv"
        };
        
        dataFilePath = alternativePaths.FirstOrDefault(File.Exists);
        
        if (dataFilePath == null)
        {
            throw new FileNotFoundException("Data file not found in any expected location");
        }
    }
}

Console.WriteLine($"Loading data from: {dataFilePath}");
var features = mlManager.LoadData(dataFilePath);

Error: "Invalid data format"

Error: Could not parse data file
?? Solution:
// Validate data before processing
try
{
    // Preview and validate data first
    var features = mlManager.ValidateAndPreviewData(dataFilePath, numRows: 5);

    {
        throw new InvalidOperationException("No valid features found in data file");
    }
    
    Console.WriteLine($"Data validation successful. Features: {string.Join(", ", features)}");
    
    // Load full dataset
    var loadedFeatures = mlManager.LoadData(dataFilePath);
}
catch (Exception ex)
{
    Console.WriteLine($"Data validation failed: {ex.Message}");
    
    // Try different file formats or cleaning steps
    // Check for:
    // - Correct CSV delimiter (comma, semicolon, tab)
    // - File encoding (UTF-8, ASCII)
    // - Header row presence
    // - Special characters in data
}

??? Assistant Class Issues

Problems specific to using the 13 assistant classes and their solutions.

Assistant Class Problems

Error: "Assistant method failed"

Exception in assistant method execution
?? Solution:
// Wrap assistant operations in try-catch blocks
if (mlManager is PythonMLManager manager)
{
    try
    {
        // Data preprocessing with error handling
        manager.DataPreprocessing.ImputeMissingValues("mean");
        Console.WriteLine("? Missing values imputed successfully");
    }
    catch (Exception ex)
    {
        Console.WriteLine($"? Missing value imputation failed: {ex.Message}");
        // Try alternative strategy
        try
        {
            manager.DataPreprocessing.ImputeMissingValues("median");
            Console.WriteLine("? Missing values imputed with median strategy");
        }
        catch (Exception ex2)
        {
            Console.WriteLine($"? Alternative imputation failed: {ex2.Message}");
            // Drop missing values as last resort
            manager.DataPreprocessing.DropMissingValues("rows");
        }
    }
}

Error: "Feature engineering failed"

Error in polynomial feature generation
?? Solution:
// Validate features before engineering
if (mlManager is PythonMLManager manager)
{
    try
    {
        // Check feature types and values
        var numericalFeatures = features
            .Where(f => IsNumericalFeature(f))
            .ToArray();
            
        if (numericalFeatures.Length == 0)
        {
            Console.WriteLine("?? No numerical features found for polynomial generation");
            return;
        }
        
        // Start with lower degree and limited features
        var selectedFeatures = numericalFeatures.Take(3).ToArray();
        
        manager.FeatureEngineering.GeneratePolynomialFeatures(
            selectedFeatures: selectedFeatures,
            degree: 2,
            includeBias: false,
            interactionOnly: true);
            
        Console.WriteLine($"? Polynomial features created for: {string.Join(", ", selectedFeatures)}");
    }
    catch (Exception ex)
    {
        Console.WriteLine($"? Polynomial feature generation failed: {ex.Message}");
        Console.WriteLine("?? Try: reducing degree, limiting features, or checking for NaN/infinity values");
    }
}

Performance Problems

?? Solutions:
// 1. Optimize data size
var features = mlManager.ValidateAndPreviewData("large_dataset.csv", numRows: 1000);

if (features.Length > 10000)

// 2. Optimize feature selection
if (mlManager is PythonMLManager manager)

}

// 3. Optimize model parameters

Issue: "Memory usage too high"

?? Solutions:
// 1. Monitor memory usage
var initialMemory = GC.GetTotalMemory(false);
Console.WriteLine($"Initial memory: {initialMemory / 1024 / 1024} MB");

try
{
    // Your ML operations
    if (mlManager is PythonMLManager manager)
    {
        // Batch processing for large datasets
        var batchSize = 1000;
        var totalRows = GetDataRowCount("large_dataset.csv");
        
        for (int i = 0; i < totalRows; i += batchSize)
        {
            // Process in batches
            var batchFeatures = LoadDataBatch("large_dataset.csv", i, batchSize);
            
            // Process batch
            manager.DataPreprocessing.StandardizeData();
            
            // Force garbage collection periodically
            if (i % (batchSize * 10) == 0)
            {
                GC.Collect();
                GC.WaitForPendingFinalizers();
                
                var currentMemory = GC.GetTotalMemory(false);
                Console.WriteLine($"Memory after batch {i}: {currentMemory / 1024 / 1024} MB");
            }
        }
    }
}
finally
{
    // Cleanup
    var finalMemory = GC.GetTotalMemory(true);
    Console.WriteLine($"Final memory: {finalMemory / 1024 / 1024} MB");
}

? Frequently Asked Questions

Common Questions

Q: Do I need to install Python separately?

A: Yes, Beep.Python.ML requires a Python installation (3.8 or higher) with the required packages. The framework uses Python.NET to communicate with your Python environment.

Q: Can I use conda instead of pip?

A: Yes, you can use conda to manage Python packages. Just ensure all required packages are installed in your Python environment.

Q: Which .NET versions are supported?

A: Beep.Python.ML supports .NET 6, 7, 8, and 9 with the same API across all versions.

Q: Why do I need to cast to PythonMLManager?

A: The IPythonMLManager interface provides core ML operations, while PythonMLManager implementation provides access to the 13 specialized assistant classes. This design maintains clean separation of concerns.

Q: Can I use custom Python scripts?

A: The framework uses a template-based approach with 101+ pre-built Python scripts. For custom functionality, you can extend the assistant classes or contribute to the template library.

Q: How do I handle large datasets?

A: Use the utility assistant's data splitting methods, apply feature selection early, consider sampling for development, and process data in batches for very large datasets.
