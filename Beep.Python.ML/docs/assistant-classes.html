13 Assistant Classes Lazy Initialization Template-Based Clean APIs

?? Overview

The Beep.Python.ML framework includes 13 specialized assistant classes, each responsible for a specific domain of machine learning operations. These assistants provide clean APIs, use template-based Python script execution, and are lazily initialized for optimal performance.

Architecture Benefits
  • Domain Specialization: Each assistant handles one specific ML domain
  • Template-Based: All operations use pre-built Python scripts with parameter substitution
  • Lazy Initialization: Assistants are created only when first accessed
  • Clean APIs: Intuitive method names with comprehensive parameter support
  • Session Isolation: Each assistant operates within isolated Python sessions
  • Error Handling: Comprehensive exception handling and logging

Here's a comprehensive example using multiple assistant classes in a complete ML workflow:

Advanced ML Pipeline with Multiple Assistants

public async Task CompleteMLWorkflowWithAssistants()
{
    // Initialize ML manager
    var mlManager = new PythonMLManager(beepService, pythonRuntime, sessionInfo);
    mlManager.ConfigureMLSessionForUser("advanced_user");

    // Load and validate data
    var features = mlManager.ValidateAndPreviewData("customer_data.csv", numRows: 10);
    var loadedFeatures = mlManager.LoadData("customer_data.csv");

    // Cast to access assistant classes
    if (mlManager is PythonMLManager manager)
    {
        // 1. Data Preprocessing Pipeline
        Console.WriteLine("Starting data preprocessing...");
        
        // Handle missing values with multiple strategies
        manager.DataPreprocessing.ImputeMissingValues("median");
        manager.DataPreprocessing.ImputeMissingValuesWithFill("bfill");
        
        // Apply scaling and normalization
        manager.DataPreprocessing.RobustScaleData();
        manager.DataPreprocessing.NormalizeData("l2");

        // 2. Feature Engineering Pipeline
        Console.WriteLine("Applying feature engineering...");
        
        // Generate polynomial features for interactions
        manager.FeatureEngineering.GeneratePolynomialFeatures(
            selectedFeatures: new[] { "age", "income", "score" }, 
            degree: 2, 
            interactionOnly: true);
        
        // Apply log transformation to skewed features
        manager.FeatureEngineering.ApplyLogTransformation(
            selectedFeatures: new[] { "income", "spending" });
        
        // Create binned features for continuous variables
        manager.FeatureEngineering.ApplyBinning(
            selectedFeatures: new[] { "age" }, 
            numberOfBins: 5, 
            encodeAsOrdinal: true);

        // 3. Categorical Encoding Pipeline
        Console.WriteLine("Encoding categorical variables...");
        
        var categoricalFeatures = new[] { "country", "product_category", "customer_segment" };
        
        // Apply target encoding for high-cardinality features
        manager.CategoricalEncoding.TargetEncode(categoricalFeatures, "churn");
        
        // One-hot encode low-cardinality features
        var lowCardinalityFeatures = new[] { "gender", "subscription_type" };
        manager.CategoricalEncoding.OneHotEncode(lowCardinalityFeatures);

        // 4. Text Processing Pipeline (if text data exists)
        Console.WriteLine("Processing text data...");
        
        if (loadedFeatures.Contains("review_text"))
        {
            manager.TextProcessing.ConvertTextToLowercase("review_text");
            manager.TextProcessing.RemovePunctuation("review_text");
            manager.TextProcessing.RemoveStopwords("review_text", "english");
            manager.TextProcessing.ApplyLemmatization("review_text");
            manager.TextProcessing.ApplyTFIDFVectorization("review_text", maxFeatures: 1000);
        }

        // 5. DateTime Processing Pipeline (if temporal data exists)
        Console.WriteLine("Processing temporal data...");
        
        if (loadedFeatures.Contains("registration_date"))
        {
            manager.DateTimeProcessing.ExtractDateTimeComponents("registration_date");
            manager.DateTimeProcessing.HandleCyclicalTimeFeatures("registration_hour", "hour");
        }
        
        if (loadedFeatures.Contains("last_login") && loadedFeatures.Contains("registration_date"))
        {
            manager.DateTimeProcessing.CalculateTimeDifference(
                "registration_date", "last_login", "days_since_registration");
        }

        // 6. Feature Selection Pipeline
        Console.WriteLine("Selecting relevant features...");
        
        // Remove low-variance features
        manager.FeatureSelection.ApplyVarianceThreshold(threshold: 0.01);
        
        // Remove highly correlated features
        manager.FeatureSelection.ApplyCorrelationThreshold(threshold: 0.95);

        // 7. Handle Imbalanced Data
        Console.WriteLine("Handling class imbalance...");
        
        // Apply SMOTE for synthetic minority oversampling
        manager.ImbalancedData.ApplySMOTE("churn", samplingStrategy: 1.0f);

        // 8. Data Quality Checks
        Console.WriteLine("Performing data quality checks...");
        
        // Remove statistical outliers
        manager.DataCleaning.RemoveOutliers(zThreshold: 3.0);
        
        // Remove duplicate records
        manager.DataCleaning.DropDuplicates();
        
        // Standardize categorical values
        var categoryReplacements = new Dictionary<string, string>
        {
            ["M"] = "Male",
            ["F"] = "Female",
            ["m"] = "Male",
            ["f"] = "Female"
        };
        manager.DataCleaning.StandardizeCategories(
            featureList: new[] { "gender" }, 
            replacements: categoryReplacements);

        // 9. Dimensionality Reduction (if needed)
        Console.WriteLine("Applying dimensionality reduction...");
        
        // Apply PCA to reduce feature space while retaining 95% variance
        manager.DimensionalityReduction.ApplyPCA(nComponents: 20);

        // 10. Data Splitting
        Console.WriteLine("Splitting data for training...");
        
        var splitPaths = manager.Utilities.SplitData(
            dataFilePath: "processed_customer_data.csv",
            testSize: 0.2f,
            validationSize: 0.1f,
            trainFilePath: "train_data.csv",
            testFilePath: "test_data.csv",
            validationFilePath: "validation_data.csv",
            primaryFeatureKeyID: "customer_id",
            labelColumn: "churn");

        Console.WriteLine($"Data split into: {string.Join(", ", splitPaths)}");
    }

    // 11. Model Training (using core ML manager)
    Console.WriteLine("Training machine learning model...");
    
    var modelId = Guid.NewGuid().ToString();
    var parameters = new Dictionary<string, object>
    {
        ["n_estimators"] = 200,
        ["max_depth"] = 15,
        ["min_samples_split"] = 5,
        ["class_weight"] = "balanced",
        ["random_state"] = 42
    };

    mlManager.TrainModel(modelId, MachineLearningAlgorithm.RandomForestClassifier, 
                        parameters, loadedFeatures, "churn");

    // 12. Model Evaluation and Cross-Validation
    Console.WriteLine("Evaluating model performance...");
    
    if (mlManager is PythonMLManager evalManager)
    {
        // Perform cross-validation
        evalManager.CrossValidation.PerformCrossValidation(modelId, numFolds: 10);
        
        // Get classification scores
        var scores = mlManager.GetModelClassificationScore(modelId);
        Console.WriteLine($"Model Performance - Accuracy: {scores.Item1:F4}, F1-Score: {scores.Item2:F4}");
        
        // 13. Create Comprehensive Visualizations
        Console.WriteLine("Generating visualizations...");
        
        evalManager.Visualization.CreateROC(modelId, "model_roc_curve.png");
        evalManager.Visualization.CreateConfusionMatrix(modelId, "confusion_matrix.png");
        evalManager.Visualization.CreateLearningCurve(modelId, "learning_curve.png");
        evalManager.Visualization.CreatePrecisionRecallCurve(modelId, "pr_curve.png");
        evalManager.Visualization.CreateFeatureImportance(modelId, "feature_importance.png");
        
        // Generate comprehensive HTML report
        evalManager.Visualization.GenerateEvaluationReport(modelId, "model_evaluation_report.html");
        
        // 14. Export Results
        Console.WriteLine("Exporting results...");
        
        evalManager.Utilities.ExportTestResult("predictions.csv", "customer_id", "churn_prediction");
    }

    Console.WriteLine("? Complete ML workflow finished successfully!");
    Console.WriteLine("?? Generated files:");
    Console.WriteLine("   - model_roc_curve.png");
    Console.WriteLine("   - confusion_matrix.png");
    Console.WriteLine("   - learning_curve.png");
    Console.WriteLine("   - feature_importance.png");
    Console.WriteLine("   - model_evaluation_report.html");
    Console.WriteLine("   - predictions.csv");
}

? Best Practices for Assistant Classes

Usage Guidelines
  • Cast to Implementation: Cast IPythonMLManager to PythonMLManager to access assistants
  • Sequential Operations: Apply operations in logical order (preprocessing ? feature engineering ? selection)
  • Session Management: Ensure sessions are configured before using assistants
  • Error Handling: Wrap assistant operations in try-catch blocks
  • Parameter Validation: Validate parameters before calling assistant methods
Performance Tips
  • Lazy Loading: Assistants are created only when first accessed
  • Session Reuse: Reuse the same session for multiple operations
  • Batch Operations: Combine related operations when possible
  • Memory Management: Dispose of managers properly after use
  • Script Caching: Template scripts are cached for performance
