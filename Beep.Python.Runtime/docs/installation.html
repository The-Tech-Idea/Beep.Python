Prerequisites

  • Python 3.8 or higher installed on your system
  • .NET SDK/Runtime matching your target framework:
    • .NET 6.0
    • .NET 7.0
    • .NET 8.0
    • .NET 9.0
  • Visual Studio 2022 or later (recommended for development)
Note: Beep.Python.Runtime.PythonNet supports multiple .NET versions (6.0, 7.0, 8.0, and 9.0) to provide flexibility for different project requirements.

Installing Python

If you don't have Python installed, follow these steps:

Windows

  1. Download the latest Python installer from python.org
  2. Run the installer and make sure to check the "Add Python to PATH" option
  3. Complete the installation with the recommended settings

Verify Python Installation

python --version
pip --version

macOS/Linux

  1. Python is often pre-installed on these systems, but you may need to install a newer version
  2. For macOS, you can use Homebrew: brew install python
  3. For Linux, use your distribution's package manager (e.g., apt install python3 on Ubuntu)

NuGet Package Installation

Install the Beep.Python.Runtime.PythonNet package using one of these methods:

Using Package Manager Console

Install-Package Beep.Python.Runtime.PythonNet

Using .NET CLI

dotnet add package Beep.Python.Runtime.PythonNet

Manually Editing Project File

<ItemGroup>
  <PackageReference Include="Beep.Python.Runtime.PythonNet" Version="1.0.43" />
</ItemGroup>

Required Dependencies

The following packages will be automatically installed as dependencies:

  • pythonnet (v3.0.3) - The core .NET-Python integration library
  • Autofac (v8.0.0) - Used for dependency injection
  • Beep.Python.Model (v1.0.16) - Data models for Python integration
  • CommunityToolkit.Mvvm (v8.2.2) - MVVM infrastructure
  • TheTechIdea.Beep.Container.Model (v1.0.0.46) - Container abstractions
  • TheTechIdea.Beep.DataManagementEngine (v2.0.6) - Data management components

Post-Installation Setup

Python Environment Configuration

The library will attempt to detect Python installations automatically, but you can also configure a specific Python environment:

Custom Python Environment Setup

// Example of custom Python environment setup
var runtimeManager = new PythonNetRunTimeManager(beepService);

// Specify a custom Python installation path
var customPython = new PythonRunTime
{
    RuntimePath = @"C:\Path\To\Your\Python\python.exe",
    Name = "Custom Python Environment"
};

runtimeManager.Initialize(customPython, null, "my-env", PythonEngineMode.SingleUser);

Setting Up Virtual Environments

For better isolation, you can create a virtual environment:

Creating a Virtual Environment

// Example of creating a virtual environment
var pythonRunTime = runtimeManager.PythonInstallations.First();
var virtualEnvManager = runtimeManager.VirtualEnvmanager;

var myEnv = new PythonVirtualEnvironment
{
    Name = "data-science-env",
    Path = @"C:\Path\To\Virtual\Environments\data-science-env",
    BaseInterpreterPath = pythonRunTime.RuntimePath
};

virtualEnvManager.CreateVirtualEnvironment(pythonRunTime, myEnv);

Troubleshooting

Common Issues

  • Python Not Found: Ensure Python is in your PATH environment variable or specify the exact path when initializing the runtime.
  • Python.NET Version Conflicts: If you experience DLL loading issues, check for multiple versions of Python.NET in your project's dependencies.
  • GIL Deadlocks: Be careful with accessing Python from multiple threads. Use the provided async methods for best results.

Diagnosing Python Environment

Environment Diagnostic Code

// Diagnostic code to check Python environment
var diagnostics = PythonRunTimeDiagnostics.CheckPythonInstallation(
    pythonPath: @"C:\Path\To\Python\python.exe");

Console.WriteLine($"Python version: {diagnostics.Version}");
Console.WriteLine($"Status: {diagnostics.Status}");

if (!diagnostics.IsValid)
{
    Console.WriteLine($"Error: {diagnostics.ErrorMessage}");
}
